    你这个问题有点泛了。

    C语言的题目分为很多种：
    1、内部处理的细节，考你对C语言本身的了解，如打印a[1]和1[a]会出现什么结果。
    这种题目，往往是面试的时候会问到，但事实上它们用处并不大。一门好的语言，应该不需要coder去理解过多的细节。
    C语言的细节是比较多的，我推荐你学习以下几本书：《C和指针》、《C陷阱与缺陷 : C语言调试指南》、《C Primer Plus》以及《编程之美》等。这些书都是公司招聘出题的首选，如果你通读了以上四本书，基本这类题就难不倒你了。

    2、算法与数据结构，Online judge题目。
    这方面，建议很简单：先通读算法与数据结构，然后再在做题中不断加深对算法的理解。书籍推荐最经典的《数据结构与算法分析》、《算法导论》，配合维基，基本上是攻无不克。
    事实上，直接用维基学习，辅以google，在我看来是最好的做法。你只要学懂Category:数据结构、Category:算法，就可以称为一个算法大牛了。
    OJ方面，建议选择POJ，UVaOJ。

    3、项目，用C语言写一个项目。（不需要深究算法，也不需要多深的C语言细节了解）
    练习C语言最实在，也是最OK的方法是写一个项目。以项目做题，你才会明白哪里有困难需要克服，你又应该学习什么。在写一个项目之前，你得弄清楚三件事：
    • 编译环境，是Linux(GNU C, GCC)，还是Windows(ANSI C, VS2010)。两者之间主要的区分在库函数上， 例如你可以调用前者的snprintf，以及POSIX的正则函数，而在ansi c中是没有的。C语言写100行代码MATLAB只用写一行，大部分原因就是因为C语言没有字符串的正则处理。
    • 最基础的C语言知识。你手头必须要有一份C语言库函数列表，包括atoi / bsearch / qsort / fgets等函数，以及它们是在哪些头文件里——你最好能清楚每一个库函数都有什么功能，免得以后费尽心思google。并且你得熟悉数组、指针、链表、枚举等（算法与数据结构的基础）的用法。
    • 想要达到的项目成果。你想要写一个文字游戏 / 3D爆炸效果 / 简单的聊天客户端，你必须要有明确的目标，去各个源代码库上寻找最佳实践。被动的等待别人教导是非常非常非常非常非常低效的。

    你该到这些地方寻找最佳实践：
    • 开源项目全集：Ohloh, the open source network
    • Github: GitHub Build software better, together.
    • Sourceforge: SourceForge - Download, Develop and Publish Free Open Source Software

    比如你可以这样搜索：Search · 聊天 · GitHub，就会发现一堆堆的工程。
    从网络游戏到聊天软件，应有尽有，随时欢迎你拿来研究。

    注：写出一个著名的开源项目，比拿了ACM世界第一屌多了。修改

    提问应遵循真实、客观、简洁、明确、规范的原则
    1、提问应尽可能简洁明了，避免「请问」「有谁知道」「谢谢！」等与问题本身无关的附加语。
    2、应避免使用「为神马」「童鞋」「肿么办」等网络用语。
    3、基于个人判断或传言的问题，在提问时应在补充说明里详细说明原由
    4、避免太宽泛问题，如调查性的问题容易变成每个人都可以发表观点，但对其他人没有价值。

    知乎禁止这些提问
    1、和已有问题完全重复的问题，为避免重复，提问前可以先进行搜索

    3、投票类问题。例如：你支持方舟子还是韩寒？
    4、作业或其他希望代为完成个人任务类的内容，以及针对个人具体病情的求医问药类问题
